# Client Zero — Build ADAMAS for Falcon Intelligence Group first

You just closed a client: yourself. Run the entire chain on FIG before any paying client.
Treat it exactly like a real engagement — same documents, same checklists, same invoices
(book it internally). Where the manuals are wrong, fix the manuals, not just the problem.

## Why this is the right order (the answer to "should we build for ourselves first?")

Yes — because it converts four open risks into assets in ~2 weeks:
1. **Product risk** → a working reference install (the v1 stack in `02-…` §3 is real,
   buildable software; Hermes slots in when its steps are documented).
2. **Credibility risk** → real screenshots for /product.html (replacing the mockups) and
   the first case study with an honest metric.
3. **Delivery risk** → every checklist in this folder gets executed once and corrected;
   you learn the true hours per stage (which validates or corrects the $5k pricing).
4. **Sales risk** → "I run my company on it" is the only claim an early-access vendor
   can make that a skeptical Mittelstand founder accepts.

## Two-week schedule

**Week 1 — Audit + Install**
- [ ] Day 1: Stage 0 paperwork on yourself (folder, mock invoice, signed-with-yourself DPA
      — yes, really; it tests the documents).
- [ ] Day 1–2: Clarity Audit self-interview using the guide in `01-…` §2 — record it,
      run the synthesis prompt, write the Clarity Report for FIG. (Hard part: be honest
      about your own knowledge traps — ADAMAS itself is a key-person risk: you.)
- [ ] Day 3: Order/repurpose the Mac Mini. Execute `02-installation-manual.md` literally,
      top to bottom, checking every box. Note every step that's unclear or wrong → fix
      the manual the same day.
- [ ] Day 4–5: Seed the FIG ledger — your 20 defining decisions exist already in raw
      form: pricing decisions, declining/choosing clients, the local-first bet, naming,
      the website plan decisions (this engagement!). Use the reconstruction prompt.

**Week 2 — Build + Handover (to yourself)**
- [ ] Day 6–8: Weekly-review habit live; process your real inbox (call notes, email
      exports) with the extraction prompt; confirm/dismiss candidates.
- [ ] Day 9: Generate FIG's onboarding asset ("Onboarding: ADAMAS delivery engineer" —
      the document a future contractor would get). Run the Build QA checklist.
- [ ] Day 10: Assemble the Handover Pack for FIG (System Passport, runbook card). Take
      the real screenshots (ledger view, inbox, asset generation) — with sample/FIG data
      you're comfortable publishing.

## Definition of done (Client Zero)

- [ ] All checklists in 00–03 executed once; every correction committed back to these files.
- [ ] FIG vault running daily; you personally do the Friday review for 4 consecutive weeks.
- [ ] /product.html mockups replaced with real screenshots (founder TODO in that file).
